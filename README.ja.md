# Mycel

これは、マインドマップツールです。データはシステムのフォルダ、ファイルの構造をベースにしてます。

ファイルを縦方向の一覧として扱うのではなく、フォルダとファイルを接続されたノードとしてホワイトボード風のキャンバスに配置します。通常のシステムフォルダとファイルをそのまま使いながら、プロジェクト構造、文書群、知識ベースを地図のように整理し探索するためのツールです。

- 現在のバージョン: 0.3.7
- 変更履歴: [CHANGELOG.md](CHANGELOG.md)
- 開発予定: [docs/development-plan.ja.md](docs/development-plan.ja.md)
- 英語版: [README.md](README.md)

## 主な機能

### マインドマップ表示

- フォルダとファイルをノードとして表示
- 広いホワイトボード風キャンバスで自由にパン/ズーム
- フォルダの折りたたみ/展開
- ファイルとフォルダを混在した順番で並び替え
- ファイル右端へのドロップによるファイル同士の関連付け
- 関連先ファイルをリンク元ファイルの右側へ配置
- 複数選択、範囲選択、範囲へのズーム

### ファイル操作

- キャンバス上から `NewFile.txt` / `NewFolder` を作成
- ドラッグ&ドロップでフォルダ間移動
- OS からファイルやフォルダをフォルダノードへドロップしてコピー
- `Ctrl + C` / `Ctrl + V` による選択項目のコピー
- ルート配下の外部ファイル/フォルダ変更を監視し、変更フォルダ単位で自動反映
- Mycel Archive Markdown 形式でのエクスポート/インポート
- F2 によるインライン名前変更
- 右クリックのコンテキストメニューから更新、削除、色設定、作成、開く

### プレビューと編集

- テキスト、HTML、Markdown、CSV、画像、動画のプレビューペイン表示
- テキストと Markdown のインラインプレビュー
- プレビュー内テキストの選択とコピー
- 画像や PDF などの簡易プレビュー枠
- 選択した編集可能ファイルをプレーンテキストで直接編集する編集モード
- プレビューペインは左、右、下へ配置切り替え可能
- プレビュー/編集モードをラベル、背景色、枠線で明確に区別
- 自動保存、`Ctrl + S` で保存、`Esc` でプレビューへ復帰
- プレビュー/編集の文字サイズを `Ctrl + マウスホイール` で変更し、次回起動後も共有して維持
- プレビュー/編集の文字サイズを `Ctrl + +`、`Ctrl + -`、`Ctrl + 0`、プレビュー/編集ペイン上のピンチ操作でも変更可能
- 画像プレビューは原寸を基本にし、長辺が 460 px を超える場合は長辺 460 px に縮小して開く
- 編集中はファイル監視による自動再描画を止め、入力フォーカスと編集状態を維持
- 1行目の `Subject: filename.txt` によるファイル名変更
- `Subject:` / `Key:` で始まる行はインラインプレビューに表示しない

### 保存される状態

Mycel は、開いたルートフォルダ配下に `.mycel` ディレクトリを作成し、ローカルメタデータを保存します。

| ファイル | 内容 |
| --- | --- |
| `.mycel/order.json` | 並び順 |
| `.mycel/colors.json` | ユーザー指定色 |
| `.mycel/previews.json` | プレビューの開閉状態とサイズ |
| `.mycel/collapsed.json` | フォルダの折りたたみ状態 |
| `.mycel/links.json` | ファイル同士の関連 |
| `.mycel/view.json` | キャンバス表示状態、ウィンドウサイズ、最大化/全画面状態 |

`--no-mycel` モードでは、これらのファイルを読み込まず、作成もしません。このモードでは並び替え、ノードの色変更、ファイル同士の関連付け、折りたたみ状態、表示状態、ウィンドウ状態の復元は無効になります。

最近使ったルートフォルダの履歴はアプリ設定に保存されます。正規化した絶対パスで重複排除し、最大 20 件まで保持します。

### アーカイブのエクスポート/インポート

ツールバーの `Export` で、開いているルートフォルダを Mycel Archive Markdown として書き出します。

- Mycel がテキストとして扱うファイルは、1つの Markdown ファイル内に本文ごと埋め込みます。
- 本文中で `` ``` `` または `\` で始まる行はエスケープし、読み込み時に元へ戻します。
- 画像、PDF、動画、Office 文書などのバイナリファイルは、相対パスを保ったまま隣接する `.assets` フォルダへコピーします。
- Markdown にはバイナリ参照、並び順、色、プレビュー状態、折りたたみ状態、ファイル同士の関連を記録します。

ツールバーの `Import` で Mycel Archive Markdown を読み込み、選択したインポート先フォルダへ復元します。既存ファイルは上書きせず、同名ファイルはスキップして結果に表示します。

## 起動

カレントディレクトリを開く:

```sh
./build/mycel
```

Windows MSVC ビルド:

```powershell
.\build-windows-msvc\mycel.exe
```

任意のディレクトリを開く:

```sh
./build/mycel /path/to/project
```

`.mycel` を使わずに開く:

```sh
./build/mycel --no-mycel /path/to/project
```

バージョンを表示して終了する:

```sh
./build/mycel --version
```

通常起動時、ルートディレクトリに `.mycel` フォルダがない場合は、`.mycel` を作成するか、`--no-mycel` モードで開くか、履歴からルートフォルダを選ぶか、別のルートディレクトリを選択するかを確認します。

## 操作

### 基本操作

| 操作 | 動作 |
| --- | --- |
| マウスホイール | ズーム |
| タッチパッドのピンチ | ズーム |
| タッチパッドスクロール | スクロール |
| Alt + 左ドラッグ | キャンバス移動 |
| 中ボタンドラッグ | キャンバス移動 |
| 何もない場所で右ボタンドラッグ | キャンバス移動 |
| 何もない場所を左ドラッグ | 範囲選択 |
| 範囲選択後に Enter | その範囲へズーム |
| Ctrl + 0 | 全体表示 |
| + / - | ズームイン/ズームアウト |
| F5 | 全体をリロード |
| F11 | ウィンドウを最大化/通常表示 |
| ? | チートシートを表示 |

macOS では、タッチパッドの2本指スライドでキャンバスを移動し、ピンチでズームします。Windows / Linux では従来のホイール動作を維持します。

### 選択とナビゲーション

| 操作 | 動作 |
| --- | --- |
| ノードを左クリック | 選択 |
| Tab | 同じ階層の次の表示項目へ移動 |
| Shift + Tab | 同じ階層の前の表示項目へ移動 |
| ↑ / ↓ | 同じフォルダ内の前/次の項目へ移動 |
| ← | 上位フォルダへ移動 |
| → | フォルダ内の最初の項目へ移動 |
| ファイルまたはフォルダを単一選択して F2 | インライン名前変更 |
| O | 選択項目を OS の既定アプリで開く |

### ファイル/フォルダ操作

| 操作 | 動作 |
| --- | --- |
| フォルダをダブルクリック | フォルダ以下を折りたたみ/展開 |
| N | 選択フォルダ、または選択ファイルと同じフォルダに `NewFile.txt` を作成 |
| Shift + N | 選択フォルダに `NewFolder` を作成 |
| ファイルをダブルクリック | プレビューを表示/非表示 |
| ファイルまたはフォルダをドラッグ | 同じフォルダ内で並び替え |
| ファイルまたはフォルダをフォルダへドロップ | そのフォルダへ移動 |
| OS からファイルまたはフォルダをフォルダノードへドロップ | そのフォルダへコピー |
| Ctrl + C / Ctrl + V | 選択項目を同じ親フォルダへコピー |
| D | 選択したファイル/フォルダの削除確認を表示 |

### プレビュー/編集

| 操作 | 動作 |
| --- | --- |
| Enter | 選択ファイルのプレビュー表示/非表示、選択フォルダの折りたたみ/展開 |
| プレビュー右下をドラッグ | プレビューサイズ変更 |
| プレビュー内の文字を選択して Ctrl + C | コピー |
| E | 選択ファイルをプレーンテキストで編集 |
| Ctrl + E | プレビューペインの表示/非表示 |
| ツールバーの Preview Place | プレビューペインを左、右、下へ移動 |
| 編集モード内の Ctrl + S | 保存し、`Subject:` によるファイル名変更を反映 |
| 編集モード内の Esc | 保存してプレビューへ戻る |
| プレビュー/編集モード内で Ctrl + マウスホイール | 文字サイズを変更 |
| プレビュー/編集モード内で Ctrl + + / Ctrl + - | 文字サイズを拡大/縮小 |
| プレビュー/編集モード内で Ctrl + 0 | 文字サイズを標準に戻す |
| プレビュー/編集ペイン上でピンチ | 文字サイズを変更 |

### 関連付け

| 操作 | 動作 |
| --- | --- |
| ファイルを別ファイルの右端へドロップ | ファイル同士を関連付け |
| 関連先ファイルを右クリックして `関連を解除` | 元のフォルダ接続へ戻す |

## ビルド

Qt 6 Widgets / Multimedia / MultimediaWidgets が必要です。

macOS:

```sh
./scripts/build-mac.sh
```

macOS のビルド出力は `build/Mycel.app` です。

Linux:

```sh
./scripts/build-linux.sh
```

Windows PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build-windows.ps1
```

Windows では、スクリプトが `C:\Qt` 配下にインストールされた Qt を自動検出できます。`C:\Qt\6.x\msvc2022_64` のような MSVC 版 Qt がある場合はそれを優先し、Visual Studio C++ 環境を取り込んで、起動時にターミナルウィンドウを開かない GUI subsystem の実行ファイルをビルドします。ビルド後は `windeployqt` で Qt ランタイムも配置します。MSVC 版 Qt を使う場合の既定ビルドディレクトリは `build-windows-msvc` です。

手動で CMake を実行する場合:

```sh
cmake -S . -B build
cmake --build build
```

### ビルドスクリプトのオプション

macOS/Linux スクリプトは以下の環境変数を参照します。

| 変数 | 内容 |
| --- | --- |
| `BUILD_DIR` | ビルドディレクトリ。既定値は `build` |
| `BUILD_TYPE` | CMake のビルド種別。既定値は `Release` |
| `CMAKE_BIN` | CMake 実行ファイル。既定値は `cmake` |
| `CMAKE_PREFIX_PATH` | Qt が自動検出されない場合の Qt インストールパス |

例:

```sh
CMAKE_PREFIX_PATH=/path/to/Qt/6.x/gcc_64 ./scripts/build-linux.sh
```

Windows スクリプトは引数を指定できます。

```powershell
.\scripts\build-windows.ps1 -BuildType Release -CMakePrefixPath "C:\Qt\6.x\msvc2022_64"
```

MinGW や特定の generator を使う場合は `-Generator` を指定します。

```powershell
.\scripts\build-windows.ps1 -Generator "Ninja" -CMakePrefixPath "C:\Qt\6.x\mingw_64"
```

### Windows インストーラー

Windows のビルド済み出力を用意したあと、Inno Setup のインストーラーを作成します。

```powershell
.\scripts\package-windows-inno.ps1
```

インストーラー作成スクリプトは `build-windows-msvc` にある既存ファイルを使い、Mycel を再ビルドしません。Inno Setup の `ISCC.exe` が必要です。標準の場所にない場合は `-IsccPath` を指定してください。

インストーラーの追加タスクで `Add Mycel to the user PATH` を選ぶと、インストール先がユーザー PATH に追加され、次回開くターミナルから `mycel` を実行できます。アンインストール時には PATH から削除されます。

## GitHub Actions

GitHub Actions 用の workflow を `.github/workflows/build.yml` に追加しています。macOS、Linux、Windows で Qt 6 をインストールし、各 OS のビルドスクリプトを実行します。
