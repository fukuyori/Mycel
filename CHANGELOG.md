# Changelog

Mycel の主な変更履歴です。

## 0.6.4

### 高速化・安定性

- `.mycel` メタデータの JSON 保存を原子的書き込み（`QSaveFile`：一時ファイル→リネーム）に変更。書き込み途中のクラッシュや電源断でも既存メタデータが破損しない。クラウド同期（Dropbox 等）が対象ファイルを掴んでリネームが失敗する場合はリトライし、最終的に直接書き込みへフォールバック（保存は必ず行われる）
- メタデータのみの操作でディスク再走査を廃止。プレビューの開閉・サイズ変更・リンクの追加/解除は、キャッシュ済みツリーの再レイアウトのみに変更（従来は毎回全ツリーを stat）。色の変更/解除は再描画のみに。大規模・クラウドフォルダで体感を大幅改善
- ノード検索（パス→NodeItem）をハッシュ索引化し O(1) に。ドラッグ/リサイズ中の毎フレーム検索を軽量化

## 0.6.3

### テキスト編集

- テキストファイルの編集をプレビューペイン内ではなく、表示中央の内蔵テキスト編集画面で開くように変更
- テキスト編集画面のフォントサイズ変更がプレビューに連動しないよう、編集用とプレビュー用のフォントサイズ設定を分離

## 0.6.2

### 横リンク（フォルダ対応）

- ファイル同士の横リンクに加え、**フォルダをファイルの右に関連づけ**られるように拡張。フォルダをファイルの右端（横リンク領域）へドロップすると、そのファイルの右にフォルダをサブツリーごと展開して接続表示する（リンク元は常にファイル、リンク先はファイル/フォルダ両対応）
- リンクされたフォルダの右クリックに「関連を解除」を追加（元の親フォルダ接続へ戻す）

## 0.6.1

### 階層型 `.mycel`（ルート化・統合）

- フォルダの右クリックに **「子ルートにする（.mycel を作成）」** を追加。指定フォルダに `.mycel` を作成し、親の `parent.json` を記録して、親からはサブルート（境界ノード）として扱う
- サブルートの右クリックに **「親に統合（.mycel を解除）」** を追加。子の `.mycel` 内メタデータ（色・並び順・プレビュー・折り畳み・横リンク）をキーを親ルート相対へ振り直して親の `.mycel` へマージし、子の `.mycel` を削除して通常フォルダへ戻す（設定は保持）

### コンテキストメニューの整理

- フォルダ／ファイルの右クリックメニューを機能グループごとに区切り線で分離し、標準的な並び（開く→作成→編集/クリップボード→整理・外観→特殊操作）に再構成

## 0.6.0

### 階層型 `.mycel`（親子で分散管理・切り替え式）

- 親フォルダをルートで開いたとき、**自前の `.mycel` を持つ子フォルダをサブルート**として扱い、フォルダアイコン＋Mycel バッジ付きの 1 ノードで表示。中身は展開しない（境界で走査を停止）
- サブルートを**ダブルクリック／Enter／右クリック「このルートを開く」**で、そのフォルダをルートに切り替え（再ルート化）。切り替え時に子の `.mycel/parent.json` へ親ルートを相対パスで記録
- 子ルートを開くと、**ルートノードの左に親ルートをキャンバス上の実ノードとして表示**し、親子を線で接続。サブルートと同一の見た目（フォルダ＋Mycel バッジ＋名前）・同一操作（クリックで選択ハイライト、ダブルクリックで親へ移動）。ズームにも追従。多段の祖先はチェーン表示
- 親ルートは子の `.mycel/parent.json`（記録）→ 無ければファイルシステム上の祖先 `.mycel` から解決
- ルート切り替え時は**ウィンドウの位置・サイズを維持**（切替先ルートの保存ウィンドウ状態は初回起動時のみ適用）
- `--no-mycel` モードではサブルート検出・切り替えを行わない
- 設計メモ: [docs/hierarchical-mycel-plan.ja.md](docs/hierarchical-mycel-plan.ja.md)

## 0.5.4

### プレビュー（PDF / EPUB のサムネイル）

- PDF の1ページ目をサムネイル表示(Qt PDF を使用、オプショナル依存)。インラインプレビューとサイドプレビューの両方で表示
- EPUB のカバー画像をサムネイル表示(`QZipReader` で ZIP からカバーを抽出。`container.xml`→OPF→`cover-image`/`meta name="cover"`/最初の画像 の順で解決)
- サムネイルは `.mycel/thumbnails/` にディスクキャッシュ(キー=パス＋更新日時＋サイズ＋解像度版)。元ファイル更新や解像度変更で自動再生成
- サムネイルは**プレビューを開いた時のみ生成**(ファイルをクリック=選択しただけでは生成しない)
- プレビュー枠を1ページ目/カバーの縦横比に合わせて表示(左右のガラ空きを解消)。サムネイルは枠の中央に表示
- 「プレビューサイズを初期化」をファイルの右クリックに追加
- 種別アイコンに PDF / EPUB バッジを追加
- 横リンク連鎖(例: `入力.txt → script.pdf → 出力.epub`)を含む行の高さを考慮してツリーを配置し、縦長プレビューが下の行に重なる問題を解消

### プレビュー（テキスト表示・リサイズ・大容量対策）

- テキストのインラインプレビューの表示行数を 7 行から最大 200 行へ拡大し、枠を広げた分だけ続きが表示されるように変更（従来は枠を広げても 7 行で切れていた）。枠の既定サイズ推定も最大 24 行まで参照
- プレビュー枠のリサイズを改善。テキストなどの通常プレビューは右下グリップのドラッグで**幅・高さを同時に自由変更**できるように変更（従来はドラッグ量の大きい片軸のみ）。画像・PDF・EPUB は縦横比を維持。自由リサイズの上限を 幅 1000px / 高さ 900px に拡大
- 大容量・大画素ファイルのプレビュー対策を追加。画像は最長辺を縮小してデコード（インライン 2560px / サイド 4096px）し、フル解像度ビットマップを読み込まないように変更。画素数が約 120MP を超える画像（小さいファイルでも巨大画素な「展開爆弾」）は読み込まずプレースホルダ表示

### メニュー / ツールバー

- メニューバーを ファイル / 編集 / 表示 / 設定 に整理
- ツールバーを「ファイル → 履歴(元に戻す/やり直す)→ 表示 → 設定」の順に整理
- ファイルメニューに「履歴から開く」(最近開いたフォルダの一覧)を追加

## 0.5.3

- ファイルの右クリックメニューに「別のアプリで開く…」を追加。Windows では OS 標準の「開く方法を選択」ダイアログを表示、macOS/Linux ではアプリ（実行ファイル）を選択して起動。既存の「開く」（OS 既定アプリ）はそのまま

## 0.5.2

- パイプライン実行を追加。スクリプト（`.py`/`.go`）ノードを右クリック →「パイプライン実行」で、そのスクリプトに接続された入力ファイル（横リンク元）を読み、出力ファイル（横リンク先、無ければ自動生成）へ書き出す。`ランナー スクリプト 入力パス 出力パス` の形で実行（`.py`→python、`.go`→`go run`）。実行後に出力を再描画して反映し、ログをウィンドウ表示

- ファイルアイコンを種別ごとに色分け。拡張子に応じたバッジ（PY/GO/TS/JS/MD/CSV/PDF/DOC/XLS/PPT/ZIP/♪/▶/⚙ など）とアクセント色で識別しやすく
- 大文字小文字だけの名前変更（例: `Script.py` → `script.py`）が「同名が既にある」と誤判定されて失敗する問題を修正（大文字小文字を区別しないファイルシステム向けに、一時名を経由して確実に変更）

## 0.5.0

### 操作履歴（取り消し / やり直し）

- 操作履歴（Undo / Redo）を新規追加。`Ctrl + Z` で取り消し、`Ctrl + Y` でやり直し。メニューバーの「編集」とツールバーの「元に戻す / やり直す」からも実行可能
- 対象操作: 移動・並び替え・名前変更・新規作成（ファイル / フォルダ）・削除・コピー貼り付け・取り込み・関連付け（追加 / 解除）・色変更 / 色クリア・折り畳み切替・プレビュー開閉・プレビューサイズ変更（=メタデータ変更を含むすべての構造操作を取り消し可能）
- 削除を完全消去から `.mycel/trash` への退避方式へ変更し、取り消しで復元できるように改善。複数選択の削除は1操作としてまとめて取り消し
- 履歴は操作前後のメタデータ全体スナップショットとファイル移動列で表現し、取り消し / やり直しで状態を正確に復元
- `.mycel/trash` は起動時と終了時に自動で掃除（履歴はセッション内のみ有効）

### 不具合修正

- 日本語など非 ASCII 名のファイル / フォルダを移動すると `std::filesystem::path` の文字コード変換失敗で例外が送出され異常終了する問題を修正（UTF-8 として解釈する `u8path` を使用）
- 複数選択した項目をドラッグしようとすると、押下時に選択が解除され単一項目しか移動できない問題を修正（複数選択中の項目を修飾キーなしで押しても選択を維持）
- 選択枠 / ホバー枠がノード枠の外側に描画されるのに `boundingRect()` が枠ぴったりだったため、選択解除後も枠が残る再描画の取り残しを修正
- ファイル / フォルダを右クリックすると選択枠が消える問題を修正（右クリック時に選択を維持してコンテキストメニューを表示）
- Windows の `QFileSystemWatcher` が監視中フォルダのハンドルを保持し、移動 / 削除（退避）/ 取り消しが「アクセスが拒否されました」で失敗する問題を修正（構造的なファイル移動の前に監視を一時解除）

## 0.3.0

- Mycel Archive Markdown 形式で、フォルダ構造、テキスト本文、並び順、色、プレビュー状態、折りたたみ状態、関連線をエクスポート/インポートできる機能を追加
- テキスト形式のファイルは1つの Markdown ファイル内にまとめ、バイナリファイルは相対パスを保った `.assets` フォルダへコピーするように追加
- マウス操作を整理し、ダブルクリックをフォルダの折りたたみ/展開とファイルのプレビュー表示/非表示に変更
- `O` キーで選択項目を OS の既定アプリで開けるように追加し、コンテキストメニュー項目を整理
- 右ペインをプレビューペインに変更し、選択ファイルのテキスト、HTML、Markdown、CSV、画像、動画を表示できるように追加
- `E` キーでテキスト、HTML、Markdown、CSV をプレーンテキスト編集し、`Esc` でプレビューへ戻るように変更
- 選択ファイルのプレビュー表示/非表示と選択フォルダの折りたたみ/展開を、`V` キーから `Enter` キーへ変更

## 0.2.6
