# Changelog

Mycel の主な変更履歴です。

## 0.9.4

### Qt private API 依存の削除

- EPUB カバー抽出を Qt private API の `QZipReader` から libzip に置き換え、Linux パッケージの `qt6-base-private-abi` 厳密依存を削除した
- libzip が見つからない環境では EPUB カバープレビューのみ無効化し、アプリ本体はビルドできるようにした

## 0.9.3

### N / Shift+N によるファイル・フォルダ作成

- 折りたたまれたフォルダを選択した状態で `N`（新規ファイル）または `Shift+N`（新規フォルダ）を実行すると、フォルダが展開されないまま新規アイテムがモデルから除外され、フォーカス移動が行われない不具合を修正した
- 作成先フォルダが折りたたみ状態の場合は自動的に展開してから再構築するようにし、新しく作成したファイル・サブフォルダへ確実にフォーカスが移動するようにした

## 0.9.2

### サイドパネルのテキストプレビュー

- サイドパネルのテキストプレビュー欄をクリックした際、ドラッグ操作の途中でも常にモーダルの編集ダイアログが開いてしまい、プレビュー内のテキストを選択してコピーできない不具合を修正した
- クリック位置からほぼ動かさずに離した場合（かつ選択中のテキストが無い場合）のみ編集ダイアログを開くようにし、ドラッグしてテキストを選択した場合はそのままコピー（Ctrl+C）できるようにした

### ボード上のノード内テキストプレビュー

- ボード上のノードに埋め込まれたテキストプレビューをクリックすると、その都度キーボードフォーカスがボード側へ移ってしまい、ドラッグでテキストを選択した直後に `Ctrl+C` を押してもノードのコピーとして扱われ、テキストをコピーできない不具合を修正した
- クリックしたプレビュー側にフォーカスを戻すようにし、選択したテキストを `Ctrl+C` で正しくコピーできるようにした
- プレビューは読み取り専用のため、フォーカスしていても `Ctrl+C`／`Ctrl+A` がボードの「ノードのコピー」「全ノード選択」ショートカットとして扱われてしまう問題を修正し、プレビュー内テキストの選択中コピー・全選択が正しく機能するようにした

## 0.9.1

### 名前順の並べ替え

- フォルダ直下の「名前順（昇順）」「名前順（降順）」を、ロケールとファイル名内の数値を考慮する自然順へ変更した（例: `item 1` → `item 2` → `item 10`）
- 大文字小文字を区別しない比較で同順位になる名前にも決定的な順序を付け、日時順で更新日時が同じ場合の名前比較にも同じ規則を適用した

## 0.9.0

### ノード検索（Ctrl+F）

- ファイル名・フォルダ名・ルート相対パスを対象にしたノード検索を追加した（設計: `docs/search-feature-design.ja.md`）
  - `Ctrl+F` または表示メニュー「検索」でツールバー直下に検索バーを表示。モーダルにせず、検索中もキャンバス操作・選択・プレビュー・編集はそのまま使える
  - 大文字小文字・全角半角（NFKC）を区別しない。空白区切りの複数語はすべて含む項目に一致（AND）
  - 並び順は 名前の完全一致 → 前方一致 → 部分一致 → 相対パスのみ一致。同順位は相対パス昇順
  - 入力中は一致ノードの強調（琥珀色、ライト/ダーク両テーマ対応）と件数表示のみを更新し、選択とプレビューは変更しない。`Enter` / `Shift+Enter` または前へ/次へボタンで結果を単一選択して画面内へ移動（循環あり）
  - 折りたたみフォルダ内・深さ制限超・1フォルダ表示件数超の結果も一時展開して表示。保存される折りたたみ状態や `order.json` は変更せず、`Esc` で検索を閉じると元の表示に戻る
  - 索引は起動時走査・バックグラウンド走査から構築し、アプリ内での作成・削除・移動・名前変更を即時反映。走査完了前は表示中ノードを対象に暫定検索し「索引作成中…」を表示
  - ボードモードでは表示中カードのみを対象に検索（非表示・未配置カードは変更しない）。ツリー/ボード切り替え時は検索語を維持して再評価
  - ルート切り替えで検索を終了し索引を破棄。検索語・結果は保存しない（`--no-mycel` でも使用可）

### 本文検索

- テキストファイルの本文も検索対象にした（名前検索と同じ検索バーに統合）
  - 対象はテキストプレビュー対象拡張子（txt/md/ソースコード/json/csv など）かつ 1MiB 以下のファイル。クラウドプレースホルダーは検索のためにダウンロードしない
  - 本文の読み取りと照合はバックグラウンドスレッドで実行し、GUI を止めない。検索語の変更で古い走査はキャンセル
  - 読み取った本文はサイズ・更新日時をキーにメモリ上へキャッシュし、2回目以降の検索を高速化。ルート切り替えで破棄
  - 名前・パスに一致しなかった本文のみの一致は結果の末尾に並び、検索バーに「（本文一致）」と表示して区別
  - 走査中は「本文検索中…」を表示し、完了すると件数と結果に本文一致を反映

## 0.8.1

### 外部ルートリンク（親フォルダ外の子ルート）

- 親フォルダの外にあるルートを、扉ノードとして親のマップにリンクできるようにした
  - 作成: フォルダ（ルート含む）右クリック「外部ルートをリンク…」→ フォルダ選択。現在のルート内・重複・存在しないパスは拒否
  - 解除: 扉ノード右クリック「外部ルートのリンクを解除」。記録のみ削除し、リンク先の実フォルダには一切触れない
  - 保存先は親ルートの `.mycel/external-roots.json`。同一ドライブなら相対パスで記録し、親ツリーごと移動しても追従。リンク先が見つからない間は扉を表示せず、復活すると再表示
  - 扉は物理サブルートと同じ見た目・操作（mycel バッジ、Enter/Space/ダブルクリック/「このルートを開く」で切り替え、親ルートバーで復帰）。並び替えも通常どおり
  - 作成・解除とも Ctrl+Z で戻せる
  - 保護: 外部扉ノードへの削除・移動・リネーム・コピー・横リンク・内部への新規作成はブロック（リンク先を誤って壊さない）

### スキャン境界の厳格化

- 再スキャン類（ファイル監視の登録・起動スキャン・2分間隔のバックグラウンド整合性チェック・ルート切替時のハッシュ更新）が、サブルート（子ルート）の内部まで潜っていたのを境界で停止するようにした。各ルートのチェックはそのルート内で完結し、親を開いている間に子ルートの中身を走査・監視・ハッシュ計算しない（子を開いている間も親側はチェックしない。従来から親は `parent.json` の参照のみ）

## 0.8.0

### キーボードショートカットの拡充

- 新規ショートカットを追加
  - `1`〜`6` : 選択項目にカラーパレットの色をワンタッチ割り当て
  - `0` : 選択項目の色をクリア
  - `Shift+D` : 関連(横リンク)を解除
  - `Shift+O` : 選択ファイルを別のアプリで開く…
  - `Shift+↑` / `Shift+↓` : 表示順で選択範囲を拡大・縮小(キーボードでの複数連続選択)
  - `Ctrl+↑` / `Ctrl+↓` : 兄弟内で並び順を変更(横リンクのターゲットはリンク内で上下移動)
  - `Ctrl+←` : 選択項目を上位フォルダへ移動(横リンクのターゲットは接続を1レベル上へ変更。連鎖の最上位では関連解除)
- 既存キーの再編
  - `Enter` : ファイル/フォルダの名前変更に変更(`F2` も引き続き利用可)
  - `Space` : 従来 `Enter` が担っていたファイルのプレビュー表示/非表示・フォルダの展開/折りたたみ・サブルートへの切り替えを引き継ぎ
  - `Shift+N` : `N` と同じ規則に統一。フォルダ選択時はその内側に、ファイル選択時は同階層に新規フォルダを作成
- テキスト編集ダイアログに `Ctrl+W`(閉じる)と `Ctrl+Enter`(保存して閉じる)を追加

### テキストエディタの編集補助（ダイアログ・サイド編集ペイン共通）

- 自動インデント: 改行時に前の行のインデント(スペース/タブ)を引き継ぐ
- 箇条書きの自動補完: `- ` `* ` `+ `、番号付き(`1.` `1)` は次番号に自動加算)、チェックボックス(`- [ ]`)、引用(`> `)の行で改行すると次の行にマーカーを自動挿入。マーカーだけの空行で Enter を押すとマーカーを消してリストを終了
- `Shift+Enter` : マーカー補完なしのプレーン改行(インデントのみ維持)
- `Tab` / `Shift+Tab` : 複数行選択時に選択行をまとめてインデント/解除(Shift+Tab は単一行でも可)
- 複数選択時のコンテキストメニューに「関連を解除」「プレビューサイズを初期化」「色」を追加(選択全体にまとめて適用、Undoは1回で戻る)

### コンテキストメニューの見た目・操作性改善

- 選択できない(無効化された)コンテキストメニュー項目が、通常の項目と同じ色で表示され区別がつかなかった問題を修正。無効な項目は薄い色で表示されるようにした

## 0.7.8

### 横リンクの連鎖レイアウトの余白過多を修正

- 0.7.6で連鎖の重なりを解消した際、対処の副作用として、連鎖を持つリンク先の下に本来より大きな空白ができてしまっていた不具合を修正。次のノードまでの間隔を「連鎖の高さ全体」ではなく「実際に下方向へ必要な半分」だけ予約するようにした

### ルート切り替え時のバックグラウンドスキャン競合を修正

- 起動直後やバックグラウンド整合性チェックの最中にルート(フォルダ)を切り替えると、切り替え前のルートに対するスキャン結果が非同期に届いた際、切り替え後の新しいルートの状態に誤って適用されてしまう可能性があった不具合を修正。スキャン結果を適用する前に、対象ルートが現在開いているルートと一致するか確認するようにした

### `resetFileSystemWatcher()` のワーカースレッド化

- 再構築やボードモード切替のたびに実行され、ツリー全体を同期的に再帰走査していたファイル監視パスの再登録処理を、ワーカースレッドで行うように変更(0.7.5でバックグラウンド整合性チェックに行ったのと同じ対応)

## 0.7.6

### 横リンクの連鎖(リンク先がさらに別のリンク元になる場合)の表示崩れを修正

- リンク元を配置順(パス名の辞書順)で処理していたため、あるリンク先(A)がさらに別のリンク(A→B)の元でもある場合、Aの表示位置が確定する前にA→Bの配置を計算してしまうことがあり、Bが誤った位置に配置され、線が無関係なノードを横切って表示される不具合を修正。リンク元の処理順を、連鎖の根本から辿る依存順に変更した
- 横リンクのスタック内で、ターゲットがファイルかつ自身も別のリンクの元になっている場合に、その先の連鎖分の高さが予約されていなかった不具合を修正。フォルダターゲットでは連鎖分の高さを予約していたのに対し、ファイルターゲットだけ考慮が漏れていたため、連鎖を持つリンク先の表示が隣のノードの表示と重なっていた

## 0.7.5

### 不要なツリー全体再走査の削減

- ノードのドラッグによる並び替え確定時、ツリー全体をディスクから再走査していたのをやめ、対象ディレクトリのキャッシュ済み子ノードをその場で並べ替えるだけに変更
- 大きなツリーを開いた際の自動折りたたみ判定結果を `.mycel/collapsed.json` に即時保存するように変更。判定に必要なツリー全体走査が、初回のみで済むようになった(保存前は起動・ルート切替のたびに毎回走査していた)
- 2分間隔のバックグラウンド整合性チェック(外部リネーム検出・ハッシュ更新)が、名前に反してGUIスレッド上で同期的にツリー全体を走査・ハッシュ計算していたのをワーカースレッドに移動。大きい、または低速なネットワークドライブ上のルートで発生していた周期的なUIフリーズ(Linux環境での「応答なし」表示を含む)を解消

### 横リンクのターゲット重複による表示崩れを修正

- 1つのリンク先(子)に複数のリンク元(親)から横リンクが張られた状態になると、リンク先ノードの表示位置がリンク元ごとに上書きされ、負けた側のリンク元から無関係なノードを横切って遠く離れた位置まで線が伸びる不具合を修正。「リンク先は常に1つのリンク元にのみ属する(リンク元は複数のリンク先を持てる)」という前提を、新規作成時だけでなく `.mycel/links.json` 読み込み時・外部リネーム検出によるリンク付け替え時にも適用するようにした

## 0.7.4

### 横接続（関連付け）の表示・並び替え修正

- 複数のファイルにそれぞれ横リンクを設定した際、別々のリンク元から伸びる表示同士が重なり、下側にあるはずのサムネイルが上側の表示に隠れてしまう問題を修正
- 横リンクされたノードの描画順を実際の縦位置順に揃え、万一表示が重なった場合でも下にあるノードが正しく手前に描画されるようにした
- 横リンクされたノード（ファイル・フォルダとも）の右クリックメニューに「横リンクを上へ移動」「横リンクを下へ移動」を追加し、同じリンク元にぶら下がる横リンク同士の並び順を変更できるようにした

## 0.7.3

### ファイル監視の見直し

- 監視パスの登録を「全解除→全登録」から差分方式に変更。再構築のたびに発生していた再登録コスト(1万ファイル規模でローカルでも約0.5秒)がほぼゼロになった
- 表示に変化のない変更通知(クラウド同期クライアントによる mtime タッチ等)の処理で、ツリー全体の再走査・全再登録を行わず、通知のあったパスの監視だけを張り直すように変更。Dropbox 配下のルートで約0.5秒ごとに発生していた常時負荷が実質なくなった
- ネットワークファイルシステム(SMB/NFS/AFP/WebDAV 等、Windows のネットワークドライブ含む)上のルートでは OS のファイル監視を使わず、2分間隔のバックグラウンドスイープが外部変更の反映も担当するように変更(macOS ではネットワークマウントに変更通知が届かず、Windows では毎秒の全パスポーリングに退化するため)。外部変更の反映は最大2分遅れるが、アプリ内操作の反映は従来どおり即時
- 環境変数 `MYCEL_NO_WATCHER=1` で任意の環境でスイープ専用モードを強制可能(変更通知が不安定なファイルシステム向け)

## 0.7.2

### ツリー走査の一本化

- 全ツリーを対象とする処理(ファイル監視の再登録・外部リネーム照合・ハッシュ更新)が、それぞれ個別にツリーを再帰走査していたのを、1回の共有走査に統合した
- 再構築のたびに実行される監視パス再登録は2回→1回、2分間隔のバックグラウンドスイープは約3回→1回、手動リフレッシュのメタデータ照合は約3回→1回の走査になり、ネットワークドライブ上での stat 往復が大幅に減った
- ハッシュ更新はディレクトリごとの再列挙をやめ、走査時に取得済みのファイル情報を直接使うようにした
- `--no-mycel` モードではメタデータ照合用の走査自体を省略するようにした

## 0.7.1

### 起動の高速化

- ウィンドウを即座に表示し、初回スキャンは表示後に実行するように変更。ネットワークドライブ上やファイル数の多いルートでも、起動直後に画面が開くようになった（スキャン完了までは「読み込み中…」を表示）
- ルート以下の全ツリー走査と、リネーム検出用コンテンツハッシュの生成・更新をバックグラウンドスレッドへ移動。起動時のファイル内容読み込みが画面をブロックしなくなった
- 起動時に重複していたツリー走査（監視パス収集・ハッシュ生成・照合で最大6回）を1回の走査に統合
- アプリ終了中に外部でリネームされたファイルのメタデータ引き継ぎは、バックグラウンドスキャン完了時に実行し、結果を自動で画面へ反映するように変更

### ビルド修正（macOS）

- 複数の Qt（公式インストーラ版 + Homebrew 版）が併存する環境で、任意モジュール（QtPdf など）だけが別の Qt から解決・混在リンクされ、起動時に dyld エラーで落ちる問題を修正。任意モジュールは Qt6 本体と同じインストールからのみ検索するようにした

## 0.7.0

### 外部リネーム・移動の追従（コンテンツハッシュ照合）

- Mycel を介さずに（エクスプローラーやクラウド同期クライアントなどで）ファイル名が変更・移動された場合でも、色・関連付け・プレビュー・並び順などのメタデータが新しいパスへ自動的に引き継がれるようになった
- ファイル内容の SHA-1 ハッシュを `.mycel/hashcache.json` にキャッシュし、「消えたファイル」と「現れたファイル」を内容で突き合わせて同一ファイルを認識する
- 照合はアプリ起動時・フォルダ切り替え時・ファイル変更通知時・2分間隔のバックグラウンドスイープの各タイミングで実行。フォルダをまたいだ移動も全体スイープで追従する
- 10MB 超のファイルと、クラウド上にのみ実体があるプレースホルダーファイル（OneDrive ファイルオンデマンド等）はハッシュ対象外（強制ダウンロードを避けるため）
- どこにも一致先が見つからないメタデータは削除された実体の残骸とみなして自動削除する

### 関連付け（横接続）とフォルダ移動の連動

- 別フォルダにあるファイルへ関連付けした場合、関連先ファイルを関連元のフォルダへ実際に移動してから接続するように変更
- 関連元ファイルを別フォルダへ移動すると、関連付けされた先のファイル（連鎖含む）も一緒に移動するように変更
- いずれも Undo/Redo に対応

## 0.6.9

### 関連付け（横接続）解除

- 関連付け（横接続）しているファイル・フォルダを、実体が置かれている元のフォルダへドラッグ＆ドロップすると、関連付けが解除されるように変更。複数選択したままのドラッグでも、元フォルダへ戻された項目はそれぞれ関連付けが解除される
- 解除は Undo/Redo に対応（Ctrl+Z / Ctrl+Y）

## 0.6.8

### ボードモード（カード自由配置）
