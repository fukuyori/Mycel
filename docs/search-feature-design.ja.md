# ノード検索機能 設計書

> **実装状況**: 初期実装(§12 フェーズ1〜6)と本文検索(§13)を実装済み。検索コアと本文走査は `src/search_controller.h`(単体テスト `tests/search_controller_test.cpp`)、UI・索引同期・本文走査スレッドは `src/main_window_search.cpp`。`ツリーで表示`・検索履歴などの将来拡張は未実装。

## 1. 目的

大きなマップで目的のファイルまたはフォルダをすばやく見つけ、既存の選択・プレビュー・画面移動の操作へ自然につなげる。

検索はノードを探して移動する機能とし、該当しないノードを隠す `Key:` フィルターとは分離する。検索中もマップの構造、通常の選択、プレビュー、編集を維持する。

## 2. 初期実装の範囲

### 対象

- 開いているアクティブルート内のファイル
- 開いているアクティブルート内のフォルダ
- ファイル名、フォルダ名、ルートからの相対パス
- 折りたたみフォルダ、表示深さ制限、1フォルダの表示件数制限の内側にある項目

### 対象外

- テキストファイルの本文
- `Key:` メタデータによる絞り込み
- PDF、Office文書、画像などからの文字抽出
- 正規表現、ワイルドカード、除外条件
- サブルートおよび外部ルートリンク先の内部
- 検索履歴の保存

サブルートと外部ルートリンクは扉ノード自体を名前検索の対象にできるが、その内部は現在のスキャン境界どおり、対象ルートへ切り替えてから検索する。

## 3. UI仕様

### 3.1 検索バー

検索バーはツールバー直下に配置し、通常時は非表示にする。モーダルダイアログにはせず、検索中もキャンバスを操作できるようにする。

```text
検索: [ 業務フロー                    ]  3 / 12
      01_現状把握/業務フロー.md
```

表示要素:

- 検索入力欄
- 現在位置と総件数
- 現在結果のルート相対パス
- 前へ／次へボタン
- 閉じるボタン
- 全体索引の準備中であることを示す状態表示

検索結果がない場合は `0 件` と表示する。空文字列では検索結果を作らず、強調表示も行わない。

### 3.2 キーボード操作

| 操作 | 動作 |
| --- | --- |
| `Ctrl + F` | 検索バーを開き、入力欄へフォーカスする。既に開いている場合は入力文字列を全選択する |
| 入力 | 150〜250msのデバウンス後に結果と強調表示を更新する |
| `Enter` | 次の結果を選択して表示する |
| `Shift + Enter` | 前の結果を選択して表示する |
| `Esc` | 検索を終了して検索バーを閉じる |

最後の結果から `Enter` を押した場合は最初へ、最初の結果から `Shift + Enter` を押した場合は最後へ循環する。

### 3.3 選択とプレビュー

- 入力中は一致ノードの強調と件数だけを更新し、選択とプレビューを自動変更しない。
- `Enter`、`Shift + Enter`、前へ／次へボタンを操作した時点で現在結果を単一選択する。
- 選択変更は既存の `SelectionController` を通す。
- プレビューペインが表示されている場合は、通常の選択変更と同じ経路でプレビューを更新する。
- 現在結果は `BoardView::ensureVisible` 相当の既存処理で画面内へ移動する。
- 検索終了後も最後に選択した結果は選択状態のまま残す。

入力中に選択を変更しないのは、画像、動画、HTMLなどのプレビュー読み込みが文字入力ごとに発生することを防ぐためである。

### 3.4 強調表示

- 一致している表示中ノードは、通常の選択色とは異なる淡い背景または枠で強調する。
- 現在結果は通常の選択表示を優先する。
- ライトテーマとダークテーマの両方で、選択、ドラッグ先、リンク先の表示と区別できる色を使う。
- `QGraphicsEffect` は各ノードに追加せず、`NodeItem::paint()` 内で検索状態を描画する。

`NodeItem` には表示専用の状態を持たせる。

```cpp
enum class SearchMark {
    None,
    Match,
};
```

現在結果は検索専用状態を増やさず、既存の選択状態で表現する。

## 4. 検索方式

### 4.1 採用方式

起動時およびバックグラウンド更新時の全体走査結果から、メモリ上に検索索引を構築する。

表示中の `scene_.items()` だけを検索すると、次の項目を見つけられないため採用しない。

- 折りたたみフォルダ内の項目
- `MaxDepth` より深い項目
- `MaxChildren` より後ろの項目
- ボードで非表示または未配置の項目

検索入力ごとにファイルシステムを再帰走査する方式も、大規模ルート、クラウド同期フォルダ、ネットワークドライブで入力を止めるため採用しない。

### 4.2 索引データ

```cpp
struct SearchEntry {
    QString path;
    QString relativePath;
    QString name;
    QString normalizedName;
    QString normalizedRelativePath;
    bool isDirectory = false;
};
```

索引はアクティブルートごとに保持し、ルート切り替え時に破棄する。検索結果は `SearchEntry` のコピーではなく索引位置または安定したパスで保持する。

パスは既存コードと同じ正規化済み絶対パスを識別子とし、UIにはルート相対パスを表示する。

### 4.3 索引の構築と更新

- 起動時の `StartupScanResult::directories` と `StartupScanResult::files` から構築する。
- 全体走査が完了する前でも、現在表示中のノードだけを暫定検索できるようにする。
- 全体走査中は検索バーに `索引作成中…` と表示する。
- 起動時走査が完了したら、同じ検索語で結果を再評価する。
- バックグラウンドreconcileの走査結果でも索引を置き換える。
- アプリ内での作成、削除、移動、名前変更は、操作完了後に対象項目を索引へ反映する。
- 外部変更はファイル監視またはバックグラウンド走査の結果で反映する。
- 走査開始時のルートと現在のルートが異なる結果は、既存処理と同様に破棄する。

初期実装では数万件規模を想定し、索引全体に対する線形検索を許容する。計測して問題が確認されるまでは、トライ木や転置索引を導入しない。

## 5. 検索文字列の処理

### 5.1 正規化

名前、相対パス、検索語に同じ正規化を適用する。

```cpp
text.normalized(QString::NormalizationForm_KC).toCaseFolded()
```

これにより、英字の大文字小文字と、互換文字の一部を区別せずに検索できる。パス区切り文字は表示プラットフォームにかかわらず、索引内では `/` にそろえる。

### 5.2 複数語

検索語を空白で分割し、すべての語が名前または相対パスに含まれる場合に一致とする。

```text
現状 業務
```

この例では、正規化した検索対象に `現状` と `業務` の両方を含む項目を結果にする。

### 5.3 並び順

結果は次の優先度で安定ソートする。

1. 名前の完全一致
2. 名前の前方一致
3. 名前の部分一致
4. 相対パスだけの一致
5. 同順位では相対パスの大文字小文字を区別しない昇順

初期実装では拡張子を除いた名前の完全一致を特別扱いしない。必要性を確認してから追加する。

## 6. ツリー表示での結果移動

### 6.1 一時展開

折りたたみ状態を保存する `collapsedPaths_` は検索のために変更しない。検索セッション専用の状態を追加する。

```cpp
QSet<QString> searchRevealPaths_;
QString searchCurrentPath_;
```

現在結果へ移動するときは、ルートから結果までの親フォルダを `searchRevealPaths_` に入れ、その経路を一時的に表示する。

検索を閉じたときは `searchRevealPaths_` をクリアして再描画する。永続化された折りたたみ状態は変化しないため、検索前の表示へ戻る。

検索中にユーザーが手動でフォルダを展開または折りたたんだ場合は、通常どおり `collapsedPaths_` を更新する。検索による一時展開とユーザー操作を同じ集合で管理しない。

### 6.2 表示制限を越える結果

現在の表示ツリーは、折りたたみだけでなく深さと子数にも上限がある。`scanTree()` または後継のツリー構築処理へ、一時的に必ず含めるパスを渡す。

- 現在結果へ至る経路では `MaxDepth` を一時的に越えて再帰する。
- 現在結果へ至る子が `MaxChildren` より後ろにある場合、その子を通常表示分に追加する。
- 強制追加した項目を `.mycel/order.json` や折りたたみ状態へ保存しない。
- 検索終了後の再描画で通常の表示制限へ戻す。

再描画後は `nodeItemsByPath_` から対象 `NodeItem` を取得し、`SelectionController` で選択して画面内へ移動する。

## 7. ボード表示での検索

初期実装では、現在のボードパターンで表示中のカードだけを結果移動の対象にする。

- 表示中カードは名前と相対パスで検索し、ツリーと同じ強調・次／前操作を使う。
- フォルダはボードに存在しないため、ボード内検索結果には含めない。
- 非表示カードおよび未配置カードを検索のためだけにパターンへ追加しない。
- カードの表示状態や位置を変更しない。

全体索引には非表示・未配置ファイルも含まれるため、将来は検索バーに `ツリーで表示` 操作を追加できる。初期実装では、ボードモードの結果集合を表示中カードへ限定し、結果件数の意味を明確にする。

## 8. 検索状態のライフサイクル

検索状態は次の場合に終了または再評価する。

| 事象 | 動作 |
| --- | --- |
| `Esc`、閉じるボタン | 検索語、結果、強調、一時展開を破棄する |
| ルート切り替え | 検索を終了し、旧ルートの索引を破棄する |
| ツリー／ボード切り替え | 検索語は維持し、新しい表示モードの対象で再評価する |
| ファイル追加・削除・名前変更・移動 | 索引更新後に同じ検索語で再評価する |
| 現在結果の削除 | 次の結果へ進む。結果がなければ現在位置を未選択にする |
| 全体再描画 | パスから検索強調を再適用する |

検索語、現在位置、検索結果、一時展開状態は保存しない。アプリ再起動時およびルート再オープン時は通常表示から始める。

## 9. 責務分担

### `SearchController`（新規）

- 索引の保持と置き換え
- 検索語の正規化
- 一致判定と並び順
- 現在位置、次、前、循環の管理
- 表示モードに応じた候補集合の受け取り

Qt Widgetsや`QGraphicsScene`には依存させず、単体テスト可能にする。

### `MainWindow`

- 検索バーの生成と表示
- 起動時・バックグラウンド走査結果の索引への受け渡し
- ツリー／ボードに応じた検索対象の指定
- 一時展開と対象ノードの再生成
- `SelectionController`、プレビュー、画面移動への接続
- シーン再描画後の強調再適用

### `NodeItem`

- 検索一致状態の保持
- テーマに応じた強調表示の描画

検索条件の解釈や結果順序は持たせない。

## 11. テスト方針

### 11.1 単体テスト

- 空文字列は0件
- 大文字小文字を区別しない
- 全角・半角を正規化する
- ファイル名の完全一致、前方一致、部分一致の順序
- 相対パスだけに含まれる語の検索
- 複数語AND検索
- ファイルとフォルダの混在
- 次／前の循環
- 索引置き換え後に現在位置が安全に更新される
- 同名ファイルが相対パスで安定して並ぶ

### 11.2 結合確認

- 折りたたみフォルダ内の結果へ移動できる
- 深さ5より深い結果へ移動できる
- 1フォルダの91件目以降へ移動できる
- 検索終了後に保存済み折りたたみ状態へ戻る
- 検索中の手動展開・折りたたみが失われない
- 結果移動でプレビューが1回だけ更新される
- 再描画後も強調が維持される
- ライト／ダークテーマで強調を識別できる
- ボードの非表示・未配置カードを変更しない
- ルート切り替え中に完了した旧索引結果を破棄する
- 外部ファイル変更後に結果件数が更新される
- `--no-mycel` モードでも検索でき、検索状態を保存しない

## 12. 実装フェーズ

1. **検索コア**: `SearchController`、正規化、複数語AND、順位付け、単体テスト
2. **基本UI**: `Ctrl + F`、検索バー、件数、次／前、選択、画面移動
3. **表示連携**: 一致ノード強調、テーマ対応、再描画後の再適用
4. **非表示経路**: 折りたたみ、深さ制限、件数制限を越えた一時表示
5. **索引同期**: 起動時走査、ファイル監視、作成・削除・移動・名前変更との同期
6. **ボード対応**: 表示中カードを対象にした検索とモード切り替え時の再評価
7. **将来拡張**: バックグラウンド本文検索、`ツリーで表示`、最近使った検索語

各フェーズで通常の選択、プレビュー、折りたたみ、ツリー／ボード切り替えが退化していないことを確認してから次へ進む。

## 13. 本文検索

> 実装済み。本文走査は `mycel::runContentSearch`(`src/search_controller.h`)、ワーカースレッド管理は `MainWindow::startContentSearch` / `finishContentSearch`(`src/main_window_search.cpp`)。本文のみ一致した結果は名前一致の後ろに並び、検索バーの相対パス表示に「(本文一致)」を付ける。

本文検索は名前検索の索引とは処理を分ける。

- 既存のテキストプレビュー対象拡張子だけを対象にする。
- 例として1MiB以下のファイルに限定し、上限は実測後に調整する。
- GUIスレッドでファイルを読まない。
- ファイルのサイズと更新日時を使って本文索引をキャッシュする。
- クラウドプレースホルダーを検索のためだけにダウンロードしない。
- 検索語変更時に古いバックグラウンド処理をキャンセルできるようにする。
- 本文一致結果には、ファイル名一致と区別できる表示を追加する。

`Key:` は本文中の文字列一致ではなく意味分類による表示絞り込みであるため、引き続き独立したフィルター機能として実装する。
