# キーボードショートカット拡充仕様

コンテキストメニュー専用だった操作へのショートカット割り当てと、既存キーの再編の確定仕様です（2026-07-10 確定・実装済み）。

## 現在のショートカット一覧（変更前）

| キー | 動作 |
| --- | --- |
| `N` | 新規ファイル作成（選択がフォルダなら内側、ファイルなら同階層） |
| `Shift+N` | 新規フォルダ作成（常に選択項目と同じ階層。未リリースの暫定変更） |
| `E` | エディタにフォーカス |
| `Enter` | プレビュー切替/展開 |
| `O` | 開く |
| `Tab` / `Shift+Tab` | 選択移動（次/前） |
| `↑` `↓` `←` `→` | ツリー内ナビゲーション |
| `D` / `Delete`（Macは`Backspace`可） | 削除 |
| `Ctrl+A`（Macは`Cmd+A`） | 全選択 |
| `Ctrl+C` | コピー |
| `Ctrl+V` | 貼り付け |
| `Ctrl+Z` | 元に戻す |
| `Ctrl+Shift+Z`（環境により`Ctrl+Y`） | やり直し |
| `F2` | 名前変更 |
| `F5` | 再読込 |
| `F11` | 最大化 |
| `F12` | デバッグパネル切替 |
| `Ctrl+E` | エディタパネル切替 |
| `Ctrl+0` | 全体フィット |
| `Ctrl+Q` | 終了 |
| `Ctrl+S` | 編集内容の保存（エディタ/サイドプレビュー編集時） |

## 確定仕様

### 新規ショートカット

| キー | 動作 |
| --- | --- |
| `Shift+D` | 関連を解除（`D`=削除との対） |
| `Shift+O` | 別のアプリで開く…（`O`=開くとの対） |
| `1`〜`6` | カラーパレットの色をワンタッチ割り当て |
| `0` | 色をクリア（プレビューサイズ初期化はコンテキストメニューのみ） |
| `Shift+↑` / `Shift+↓` | 複数連続選択。Tab/Shift+Tab の線形巡回順で範囲を広げる（実質 Tab 順の範囲選択） |
| `Ctrl+↑` / `Ctrl+↓` | 兄弟内で並び順変更（横接続のターゲットなら「横リンクを上へ/下へ移動」） |
| `Ctrl+←` | 上位フォルダに移動（構造の移動、Undo対応）。横接続の場合は上位レベルへ接続を変更 |

### 既存キーの変更

| キー | 変更後の動作 | 備考 |
| --- | --- | --- |
| `Enter` | ファイル名変更 | 従来のプレビュー切替/展開は `Space` へ。`F2` は別名として残す |
| `Space` | ファイル: ファイル名下のプレビュー表示/非表示。フォルダ: 展開/折りたたみ。サブルート: サブルートへ切り替え | 従来の `Enter` の3役をそのまま引き継ぐ |
| `Shift+N` | フォルダ選択時はサブフォルダを作成、ファイル選択時は同階層にフォルダ作成 | 暫定の「常に同階層」案をやめ、`N` と同じ規則に統一（未解決の論点を「Nに揃える」方向で解決、0.8.0） |

### 編集ダイアログ

| キー | 動作 |
| --- | --- |
| `Ctrl+W` | 閉じる |
| `Ctrl+Enter` | 保存して閉じる |

### 複数選択時のコンテキストメニュー

複数ノード選択時にも以下の項目を表示し、選択全体に適用する。

- 関連を解除
- プレビューサイズを初期化
- 色

### 実装上の注意

- 数字キーを含む新キーはすべて `textInputHasFocus()` ガードの内側（`handleBoardShortcut`）に置き、リネーム中・編集中の誤発動を防ぐ。
- ボードモードではリネーム不可のため、ボードモードの `Enter` はプレビュー切替のまま維持。`Space` も同じ動作を提供する。
- `Ctrl+←` はファイル移動を伴うため Undo 対応必須。

## 見送り・将来検討

| 案 | 動作 | 備考 |
| --- | --- | --- |
| `Ctrl+R` | パイプライン実行 / Go スクリプトとして実行 | 実行可能なファイルを選択している時のみ有効化。優先度低 |
| `H` | 非表示（パターンから隠す） | ボードモード時のみ有効。優先度低 |
| `Ctrl+→` | 前の兄弟フォルダへ格納（降格） | `Ctrl+←`（昇格）との対称。将来検討 |

- **並べ替え（名前順/日時順）**: ディレクトリ全体への操作であり、単一ノード選択のショートカットには馴染みにくい
- **子ルート化 / 親に統合**: 構造（`.mycel`の作成・解除）を変える操作のため、誤操作防止の観点から見送り
- **横リンクで作成**: 2ノードの関係（元と先）を指定する操作なので単一キーでは表現しづらい。将来的に複数選択状態での`L`（Link）キーなどを検討する余地はある
