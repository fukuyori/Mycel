# ボードモード（カード自由配置）設計メモ

ツリー表示とは別に、**フォルダを除く全ファイルをカードとして自由配置する表示モード**を追加するための設計メモ。配置は複数の「パターン」として保存・切替できる。

> **実装状況**: Phase 1〜3 実装済み（モード切替・パターン作成/名前変更/削除・初期配置＝行=フォルダ/上端揃え・自由移動＋保存・非表示/一覧からの呼び出し・カード移動/非表示/表示の Undo/Redo 統合）。

## 1. 確定仕様

### 対象と初期配置

- 対象は開いているルート配下の**全ファイル**（フォルダはカードとして表示しない）。
- サブルート（自前の `.mycel` を持つ子フォルダ）は境界で停止し、その配下は含めない。
- ツリーの折り畳み状態とは無関係に全ファイルを対象とする。
- **初期配置は「行 = フォルダ」**:
  - 第0行: ルート直下のファイル（左→右）
  - 以降、フォルダごとに1行。第1階層フォルダをツリー順に1行ずつ、その次に第2階層フォルダ… と階層順に続く
  - 各行はそのフォルダ直下のファイルを左→右（`order.json` の並び）
- 同一フォルダ内の並び順はツリーと同じ（`order.json` の並び）。
- **フォルダの見出し・区切りは表示しない**。初期配置後、カードは完全フリーに移動できる（初期配置は出発点にすぎない）。

### 配置・操作

- カードのドラッグは**完全フリー配置**（グリッドスナップなし）。作業領域は上下左右無制限。
- 移動した位置はアクティブなパターンに保存される。
- **横リンクの線は描画しない**（カードは個別に並ぶ）。
- **実ファイルの削除・リネーム・フォルダ移動は不可**（このモードでは提供しない）。
- 「削除」は行わず**「非表示」のみ**。非表示カードは位置を保持したまま隠れ、一覧から呼び出して**元の位置に再表示**できる。実ファイルは消えない。
- **画面に表示されるのは「配置済み（visible）かつ実ファイルが存在する」カードのみ**。非表示カード・新規（未配置）カードは、呼び出すまで画面上に一切表示されない。
- **後から追加されたファイルは自動配置しない**。非表示カードと同じ「一覧」に現れ、呼び出して表示する（保存位置が無い場合は現在のビュー中央に配置）。
- 内容の編集（サイドエディタ）は実ファイルへ反映する（ツリーと同じ経路）。
- プレビュー開閉・サイズ・色はツリーとメタデータを共有（`previews.json` / `colors.json`）。
- **実ファイルが消えている場合は表示からも消す**（ボードモード中もファイル監視で即時反映）。エントリ（位置）は保持するが表示・一覧掲載はせず、ファイルが戻れば一覧から呼び出せる。

### パターン

- パターン数の**上限なし**。**1 パターン = 1 JSON ファイル**（`.mycel/boards/<パターン名>.json`）。
- パターンの新規作成（現在のファイル一覧から初期配置を生成）・切替・名前変更・削除。
- パターンごとにビュー位置（中心・ズーム）を保存し、切替時に復元。

### モード・復元

- ツリー ⇄ ボードの切替 UI は表示メニュー＋ツールバー。
- 前回のモードとアクティブパターンは `view.json` に保存し、起動時に復元。
- `--no-mycel` モードでは使用不可（保存が前提のため）。

### Undo/Redo

- カードの位置変更・非表示/再表示を操作履歴（Ctrl+Z / Ctrl+Y）の対象にする。

## 2. ファイル形式（`.mycel/boards/<name>.json`）

```json
{
  "version": 1,
  "cards": {
    "<ルート相対パス>": { "x": 0.0, "y": 0.0, "visible": true }
  },
  "view": {
    "transform": { "m11": 1.0, "...": 0.0 },
    "center": { "x": 0.0, "y": 0.0 }
  }
}
```

- `cards` に**無い**ファイル = 未配置（一覧に出る。新規追加ファイルがこの状態）
- `visible: false` = 非表示（位置は保持。再表示で元の位置へ）
- 書き込みは既存の `writeJsonFileAtomic`（原子的書き込み）を使用

## 3. UI

- **表示メニュー**: 「ボード表示」トグル / 「パターン」サブメニュー（一覧・新規作成・名前変更・削除）/ 「非表示カード一覧…」
- **ツールバー**: モード切替ボタン＋パターン選択ドロップダウン
- **カードのコンテキストメニュー（ボード時）**: プレビューを開く/閉じる、非表示、色、開く、別のアプリで開く
  - ツリー用の項目（名前変更・削除・作成・貼り付け・並べ替え等）は出さない

## 4. 実装方針

- **NodeItem を再利用**: ボード用にフラットな `Node` 群（`boardNodes_`）を生成し、既存の NodeItem（カード描画・インラインプレビュー・色・選択・サイドエディタ連携）をそのまま活用する。
- **BoardView / DragController にボード分岐**: ドラッグ＝自由移動、リリースで位置をパターンへ保存（デバウンス可）。フォルダドロップ・リオーダー・リンク判定はボード時は無効。
- **コンテキストメニューにボード分岐**（NodeItem::showContextMenuAt）。
- **rebuild()/relayout() にモード分岐**: ボード時はパターンからシーンを構成。プレビュー開閉は該当カードの `syncFromNode()` のみ（全体再レイアウト不要）。
- **ファイル監視**: ボードモード中も FS 変更でボードを再構成し、実ファイルが消えたカードを即時に画面から除去する（新規ファイルは自動では出さない）。
- **履歴**: `MetadataSnapshot` にアクティブパターンの `cards` を追加し、Undo/Redo で復元。
- **ファイル列挙**: ボード用に専用スキャン（`.mycel` 除外・サブルート境界停止・折り畳み無視・`fileOrders_` 順）。

## 5. 実装フェーズ

1. **Phase 1 — 基盤**: モード切替、パターンの作成/切替/保存/復元、初期配置（行=階層）、自由移動と位置保存、ビュー状態のパターン別復元
2. **Phase 2 — 非表示/呼び出し**: コンテキストメニュー「非表示」、非表示・未配置カードの一覧ダイアログと再表示、新規ファイルの取り込み
3. **Phase 3 — 仕上げ**: Undo/Redo 統合、パターンの名前変更/削除、エッジケース（ファイル消失・復活、ルート切替との整合）
