# 階層型 `.mycel`（親子分散）モデル 設計メモ

`.mycel` メタデータを、開いた 1 つのルートに集約する現在の方式から、**サブフォルダごとに自己完結した `.mycel` を持てる「階層型（親子分散）」方式**へ拡張するための設計メモ。実装前の検討資料であり、本メモの範囲では実装は行わない。

**採用方針: 「切り替え式（ルート切り替えナビゲーション）」**。全 `.mycel` を 1 つのキャンバスに併合する「シームレス方式（最寄り `.mycel` 所有）」は、検討の結果、複雑さに見合わないため不採用とし、付録 A に代替案として残す。

> **実装状況（2026-07 時点）**: 切り替え式のコア（Phase A〜C 相当）を実装済み。サブルート検出＋境界停止、サブルートノードへの mycel アイコン表示、ダブルクリック／Enter／右クリック「このルートを開く」での下り切り替え、親ルートバー（祖先 `.mycel` チェーンをクリックして上位へ切り替え）。`--no-mycel` では無効。

## 1. 目的・背景

`mycel` のルートを複数持つフォルダ群があり、その **親フォルダをルートに設定したとき、子の `.mycel` を活かしたまま親子の階層型で分散管理したい**、という要求に応える。

狙い:

- **可搬性**: サブフォルダを別の場所へ移動しても、そのフォルダの `.mycel` が一緒に付いていくので、色・並び順・折り畳み・リンクが保持される。
- **分散運用**: サブツリー単位で独立して管理でき、それぞれを個別に Git 等でバージョン管理できる。
- **スケール**: 親が全項目の巨大な単一メタデータを抱え込まない。
- **一貫性**: 「`.mycel` があるフォルダ＝サブルート」という、ファイルシステム駆動の明快な規則。Mycel の「ファイルシステムがモデル」という設計思想に沿う。

## 2. 現状（中央集権型）と限界

### 2.1 現在の挙動

- 開いた 1 つの `rootPath_` 直下の `.mycel/` だけが唯一の保存先。`order.json` / `colors.json` / `previews.json` / `collapsed.json` / `links.json` / `view.json` と `thumbnails/`・`trash/` がすべてそこに集約される。
- メタデータのキーは「**開いたルートからの相対パス**」（`relativeKeyForPath(path) = QDir(rootPath_).relativeFilePath(path)`、`src/main.cpp`）。
- ツリー走査は全サブフォルダへ潜るが、`.mycel` という名前のエントリだけを非表示にする（ディレクトリ列挙で除外）。

### 2.2 限界

- 子フォルダが自前の `.mycel` を持っていても、**親をルートで開くと子の `.mycel` は読まれず無視される**（隠されるだけ）。親の 1 つの `.mycel` が全項目を所有する。
- 逆に子を直接開けば子の `.mycel` が使われる。つまり現状は「どちらのルートを開くか」で別々のメタデータが**排他的に**効き、両者は統合されない。
- 親の `.mycel` は子項目を**深い相対パス**で記録するため、子フォルダを移動するとメタデータが追従せず失われる。

## 3. 採用方針: 切り替え式（ルート切り替えナビゲーション）

**常にアクティブなルートは 1 つ**とし、アクティブなルートの挙動は現状の単一 `.mycel` の挙動そのままとする（メタデータの併合を行わない）。サブルートへは「切り替え」で出入りする。

### 3.1 基本規則

- **アクティブルート = 今開いている 1 つの `.mycel` ルート**。キーはそのルートからの相対パス。読み書きは現状通り。
- **サブルート = 自前の `.mycel` を持つ子フォルダ**。アクティブルートのビューでは、そのサブルートを**子フォルダ名のまま、mycel アイコンを付けて表示**する。**中身はチェックしない**（内部を列挙せず、境界で走査を止める）。
- サブルートのノードで「**このルートに切り替え（＝その mycel フォルダを開く）**」を実行すると、表示がその子へ切り替わり、そのフォルダをアクティブルートに**再ルート化**する（既存の「フォルダを開く／ルート変更」処理を再利用し、そのフォルダの `.mycel` を読み込む）。
- **子へ切り替えると、子のルートフォルダの上に、親のルートフォルダ名を mycel アイコン付きで表示する**（1 つ上のルートへ戻るための見出し）。親（その見出し）を開くと、表示は親へ戻る。多段の場合はルートチェーン（例: `親 > 子A > 孫`）として上位を辿れる。

### 3.1.1 表示仕様（具体）

- **親ビュー**: サブルート（`.mycel` を持つ子フォルダ）は、通常のフォルダノードと同じく**その子フォルダ名**で表示し、名前に **mycel アイコン**を添える。展開しても中身は出さない（サブルート＝境界）。`.mycel` を持たない普通のフォルダは従来通り展開できる。
- **子ビュー（切り替え後）**: キャンバス上部（またはパンくず領域）に、**親のルートフォルダ名＋mycel アイコン**を「1 つ上のルート」として表示する。これをクリック／開くと親ビューへ戻る。
- **戻り／上位移動**: 親ルート名の見出しが「上のルートへ」の入口を兼ねる。多段のときは親・祖父…をチェーン表示し、任意の段へ切り替えられる。既存の「履歴から開く（最近開いたフォルダ）」とも整合する。

```
親ビュー                         子ビュー（子Aへ切り替え後）
┌─────────────────────┐        ┌─────────────────────┐
│ (親ルート = 親フォルダ)      │        │ ⬢ 親フォルダ名   ← 上のルート │
│                          │  ⇄     │ ─────────────────────── │
│  📄 memo.txt             │        │ (子Aルート = 子Aフォルダ)      │
│  ⬢ 子A     ← mycelアイコン │        │   📄 note.md              │
│  ⬢ 子B                   │        │   📁 sub （普通のフォルダ）   │
│  📁 普通フォルダ（展開可）    │        │                          │
└─────────────────────┘        └─────────────────────┘
        ⬢ = mycel アイコン（サブルート／上位ルートの目印）
```

### 3.2 所有（重複が起きない）

- **親の `.mycel`** は「扉ノード（サブルートのフォルダそのもの）」の位置・色・並びだけを所有する。
- **サブルートの `.mycel`** はそのフォルダの**内部**を所有する。
- 両者の管理範囲は境界で完全に分かれるため、所有の重複や「どちらが持つか」の判定が原理的に発生しない。

### 3.3 リンク

- 横リンク（`links.json`）は**アクティブルート内で完結するものだけ**を対象とする。境界を跨ぐリンクは作成不可（またはサブルートの扉ノードを一端とするリンクのみ許可する将来拡張とする）。
- これにより「最小共通祖先（LCA）による所有」のような複雑な処理は不要になる。

### 3.4 ビュー状態

- `view.json` は各ルートが自分のものを持つため、切り替えるとそのルートのキャンバス位置・ズーム・ウィンドウ状態が復元される（現状の仕組みをそのまま活用）。

## 4. メタデータ種別ごとの扱い

| 種別 | 扱い |
| --- | --- |
| `order.json` / `colors.json` / `previews.json` / `collapsed.json` | アクティブルートのみ（現状通り）。扉ノードの並び・色は親が、内部はサブルートが各々保持 |
| `links.json` | アクティブルート内で完結するリンクのみ（現状通り） |
| `view.json` | 各ルートが個別に保持。切り替えで復元 |
| `thumbnails/` | 各ルートが個別に保持（キャッシュ）。集約不要 |
| `trash/` | 各ルートが個別に保持。削除／Undo はそのルート内で完結 |

いずれも「1 ルート = 現状の単一 `.mycel`」の枠組みをそのまま使うため、ファイル形式・スキーマの変更は不要。

## 5. 実装の追加点

現状の単一ルート機構をそのまま流用できるため、追加は限定的。

1. **サブルート検出**: 子フォルダが `.mycel` を持つ場合、その**内部を列挙せず**サブルートノード化する（ツリー走査を境界で止める＝中身はチェックしない）。現状は潜って子 `.mycel` を無視しているので、ここが唯一の挙動変更点。
2. **サブルートの表示**: 子フォルダ名のまま **mycel アイコン**を添えて表示（サブルートの目印）。
3. **「このルートに切り替え（その mycel フォルダを開く）」操作**: ダブルクリック（または右クリックメニュー）で、サブルートのフォルダへ再ルート化。既存のルート変更処理（`rootPath_` の切替＋メタデータ再読込）を呼ぶ。
4. **上位ルートの見出し**: 子へ切り替えたら、子のルートフォルダの上に**親ルートフォルダ名＋mycel アイコン**を表示し、そこから親へ戻れるようにする。多段はルートチェーン表示。既存の「履歴から開く（最近開いたフォルダ）」とも整合。
5. **ビュー復元**: `view.json` は各ルート持ちなので切替時に自然に復元される（追加実装ほぼ不要）。

### 5.1 影響範囲（確認事項）

- ディレクトリ列挙（子ノード生成）にサブルート境界での停止を追加。
- 扉ノードのダブルクリック挙動（現状はフォルダの折り畳み）を、サブルートでは「切り替え」に振り分けるか、切り替えは明示メニューに寄せるかを決める。
- ルート変更時の状態リセット（選択・ドラッグ・プレビュー等）が既存経路で正しく行われるか確認。
- `--no-mycel` モードではサブルート検出も切り替えも行わない（現状通り）。

## 6. 互換性・移行

- 既存の**単一 `.mycel`** プロジェクトは「サブルートを持たない通常ルート」として**無改修で動作**する。
- 親フォルダに `.mycel` を作って開けば、既存の子ルート群が自動的に扉ノードとして現れる（各子の `.mycel` は無改修）。明示的な移行は不要。
- 逆に、親をルートで開いたまま従来通り「子の中身も 1 枚で見たい」場合は、子に `.mycel` を作らなければ従来の一体表示のまま（サブルート化するかどうかは `.mycel` の有無で選べる）。

## 7. 段階的実装フェーズ案

1. **Phase A — 境界停止と扉ノード**: サブルート検出と、内部を展開しない扉ノード表示（バッジ付き）。この時点でツリーが境界で止まることを確認。
2. **Phase B — 切り替え**: 扉ノードからの再ルート化（下りる）と「上のルートへ」（上る）。
3. **Phase C — パンくず／履歴**: ルートチェーンのパンくず UI と切り替え。
4. **Phase D — 仕上げ**: ビュー復元の確認、選択／ドラッグ状態のリセット、`--no-mycel` 整合、エッジケース（入れ子サブルート、`.mycel` の後付け／削除）。

## 8. 未解決・要検討事項

- 扉ノードのダブルクリックを「切り替え」に割り当てるか、明示メニュー／ボタンに寄せるか（誤操作防止とのバランス）。
- サブルート内へ**ドラッグ＆ドロップで項目を移動**したときの扱い（境界を越える移動はファイル移動としては可能だが、切替前のビューでどうフィードバックするか）。
- 境界を跨ぐリンクを将来的に「扉ノード単位」で許可するか（当面は非対応）。
- 入れ子サブルートの深さ表示（パンくずの省略表現）。

## 付録 A: 代替案（不採用）— シームレス方式「最寄り `.mycel` 所有」

全 `.mycel` を 1 つのキャンバスに併合し、各項目のメタデータを「最も近い祖先の `.mycel`」が所有する方式。親を開くとサブツリー全体が 1 枚のマップに連続表示され、子ルートの設定も同一ビューに反映される。

- **所有規則**: `owningMycelDir(path)`（開いたルートを上限に親方向へ辿り最初に `.mycel` を持つディレクトリ）、`keyRelativeToOwner(path)`。
- **ロード**: 配下の全 `.mycel` を（遅延で）発見し、各ファイルをその `.mycel` 基準で共有メモリマップへマージ。
- **セーブ**: メモリマップを所有 `.mycel` ごとに仕分けし、各 `.mycel` へ相対キーで書き出す。
- **境界跨ぎリンク**: 両端の最小共通祖先（LCA）を含む最寄り `.mycel` が所有し、共通祖先基準の相対パスで保存（`owningMycelDirForLink`）。

**不採用の理由**: 併合ロード／所有仕分けセーブ／LCA リンクなど永続化層の大改修が必要で、複雑さとエッジケース（所有変更・境界跨ぎ移動・競合）が多い。「親子で分散管理したい」という要求は、より単純な切り替え式で十分に満たせる。1 枚のキャンバスで全階層を俯瞰したい需要が明確になった場合に再検討する。
